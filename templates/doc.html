  • I. HomeCloud

    • Zaloguj się na swoje konto w https://homecloud.pl oraz na swojej maszynie wirtualnej na konto -> root
      (VPS -> Akcje -> Otwórz konsolę web)
      Następnie wykonaj poniższe komendy wpisując je w terminalu.
    • Świetnie, pierwsze koty za płoty. Ale co te komendy robiły i po co były wpisywane?

      - adduser - dodanie nowego usera
      - apt install - instalację pakietu sudo - pozwoli nam on łatwo i bezpiecznie instalować niezbędne oprogramowanie
      - usermod - dodanie użytkownika ubuntu do grupy sudo
      - reboot -= restart naszego servera

    • Gdy maszyna się uruchomi po restarcie, logujemy się na dodanego użytkownika ubuntu i wykonujemy poniższe polecenia
    • Maszyna powinna już działać. Ale co to za komendy poleciały i po co ?

      - sudo apt - apt zamiennie z apt-get jest narzędziem do zarządzania naszymi aplikacjami w systemie
      - mkdir - make directory / utwórz katalog. Parametr -p tworzy całą gałąź katalogów.
      - chown - dodanie grupy i właściciela pliku
      - git - potężne narzędzie do wersjonowania kodu

  • II. Amazon AWS -> https://eu-central-1.console.aws.amazon.com

    • Zaloguj się na swoje konto w AWS aby utworzyc instancje (maszyne wirtualną)
      Przejdź na stronę EC2 wyszukując ja w dostępnej wyszukiwarce lub kliknij tutaj
      - "Launch instance" - przycisk
      - wyszukaj ubuntu w wyszukiwarce "Step 1: Choose an Amazon Machine Image (AMI)"
      - "Select" - przycisk, dla Ubuntu Server 20.04
      - pozostaw defaultową konfiguracje
      - przejdz do "Step 6: Configure Security Group" - "Add Rule" - przycisk, w utworzonym wierszu zmien Type na HTTP i Source na My IP
      - "Review and Launch" - przycisk
      - na stronie podsumowania kliknij "Launch" - przycisk
      - w oknie "Select an existing key pair or create a new key pair" - wybierz "Create a new pair", nadaj kluczowi dowolna nazwe i "Download Key Pair" - przycisk
      - utworzony plik z rozszerzeniem .pem zapisz na dysku
      - "Launch instance" - przycisk
      - na stronie Launch Status powinieneś zobaczyć komunikat "Your instances are now launching"
      - ponownie przejdź na stronę EC2 wyszukując ja w dostępnej wyszukiwarce
      - na liście powinna być podlinkowana pozycja z naszym serwerm, kliknij w link
      - strona "Instance summary for " jest podsumowaniem naszej maszyny, tu znajdziesz potrzebne informacje do kolejnych zadań

      Zajmijmy się taraz konfiguracją naszego serwera. Zaloguj się do konsoli i wykonuj poniższe polecenia.

  • III. Przydatne informacje co i jak na AWS

    • 1. Konfiguracja static IP na AWS


      Przejdź na stronę EC2 wyszukując ja w dostępnej wyszukiwarce lub kliknij tutaj
      - w lewym menu znajdź "Elastic IPs", kliknij w link
      - "Allocate Elastic IP address" - przycisk
      - pozostaw defaultową konfiguracje, "Allocate" - przycisk
      - na stronie Elastic IP addresses ustaw sobie na liscie nazwe dla IP
      - wejdź do swojego nowo utworzonego rekordu w Elastic IP
      - "Associate Elastic IP address" - przycisk
      - w polu Instance wybierz swoj serwer
      - "Associate" - przycisk
      - gratulacje, masz już stałe IP podpiete - Uwaga! Nieużywne IP może spowodować naliczenie opłaty za jego blokadę
      - skopiuj teraz IP i przejdź na strone gdzie domena została zakupiona. Znajdź miejsce gdzie możesz dopisać rekord A dla domeny, wpisz tam swoje IP.
    • https://www.youtube.com/embed/9Uu2VKoot-c

    • 2. Ustawienie hasła dostępu


      Defaultowo dostęp do konsoli bez klucza jest wyłączony i żeby aktywować taki dostęp należy ustawić hasło do konta.
      Zaloguj się do terminala i wpisz komendę: sudo passwd
      Następnie wyedytuj plik /etc/ssh/sshd_config i ustaw tam PasswordAuthentication yes
